# CONFIG <Badge type="info" text="未发布" />
设置请求配置内容，一般用于链式调用场景。
::: warning 注意事项
+ 该方法的配置内容会覆盖旧版请求配置参数内容。
:::

## 使用示例
::: tip 提示
这里示例中的内容为演示，完整的请求配置内容参考 [API设置方式](/config/request#api设置方式)
:::
```ts
request
    .config({
        loading: true,
        loadingText: '加载中...'
    })
    .post('/user/save', {
	    user_id: 1
    })
	.then((res) => {
		// 此处可自定义业务逻辑
	})
	.catch((err) => {
		// 此处仅为演示
		console.error('请求服务异常');
	});
```
